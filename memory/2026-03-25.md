## Google Contacts Blocker (14:33)
- gws always saves OAuth tokens encrypted to OS keyring (.enc)
- OpenClaw process can't access gnome-keyring even with DISPLAY=:0 / DBUS set
- All 3 auth attempts via Chrome saved .enc that can't be decrypted from OpenClaw shell
- FIX: Run `gws auth logout && GOOGLE_WORKSPACE_CLI_CREDENTIALS_FILE=~/.config/gws/credentials.json gws auth login --scopes "..."` in a REAL desktop terminal
- Hannah Seaton (ECL Designer) + Steve Lofstrom (ECL PM) saved to Fragments pending fix
- Fragment IDs: RjVsX1p0bFdOVE5YbTZQQw (Hannah), YmFXbUhYWEpxNVpFLXhTNw (Steve)
- Root cause: gws uses Rust `keyring-3.6.3` crate → stores AES-256-GCM key in `gogcli` libsecret collection
- `gogcli` collection is locked and requires GUI prompt to unlock — not accessible headlessly
- Workaround found in binary strings: "Warning: keyring access failed, falling back to file storage"
- To trigger fallback: set DBUS_SESSION_BUS_ADDRESS to invalid value before `gws auth login`
- BUT: each auth generates a new port; Chrome must redirect to THAT exact port; code is bound to port
- Regis approved ~8 auth attempts in Chrome; each code went to wrong port or got reused → all failed
- **Final state (14:53):** gws has NO credentials. ALL Google Tasks/Gmail ops broken until re-auth.
